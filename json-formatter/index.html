JSON 포맷터 / 검증기 사용 방법

API 응답, 설정 파일, 데이터 파일의 JSON을 확인하거나 정리할 때 이 도구를 사용하세요. JSON을 입력창에 붙여넣고 포맷 버튼을 누르면 들여쓰기와 줄바꿈이 적용된 읽기 쉬운 형태로 변환됩니다. 문법 오류가 있으면 오류 위치와 원인을 정확히 알려줍니다.

자주 발생하는 JSON 오류 유형 참고표

오류 유형잘못된 예시올바른 예시
작은따옴표 사용{'name': 'John'}{"name": "John"}
키에 따옴표 없음{name: "John"}{"name": "John"}
후행 쉼표 (trailing comma){"a":1, "b":2,}{"a":1, "b":2}
주석 사용{"a":1 // 주석}{"a":1}
undefined 값{"a": undefined}{"a": null}
괄호 불일치{"a": [1, 2}{"a": [1, 2]}

JSON 주요 문법 규칙

문자열: 반드시 큰따옴표("") 사용. 작은따옴표('') 사용 불가
키(key): 반드시 큰따옴표로 감싸야 함. {name: ...} 형태 불가
쉼표: 마지막 항목 뒤에 쉼표 불가 (trailing comma)
값 타입: 문자열, 숫자, boolean(true/false), null, 배열[], 객체{}
주석: JSON 표준에서 주석(//) 미지원

JSON 미니파이(압축) 용도

JSON 미니파이는 공백·줄바꿈을 제거해 데이터 크기를 최소화합니다. API 전송 시 페이로드 크기 절약, 코드에 JSON 인라인 삽입, 스토리지 공간 절약 등에 활용됩니다. Ctrl+Enter 단축키로 빠르게 포맷할 수 있습니다.

자주 묻는 질문

JSON 포맷터란 무엇인가요?
JSON 포맷터는 압축된 JSON 데이터를 들여쓰기와 줄바꿈을 적용해 사람이 읽기 쉬운 형태로 변환하는 도구입니다. API 응답이나 설정 파일 확인 시 필수적으로 사용됩니다.
JSON 유효성 검사 방법은?
JSON에 문법 오류(따옴표 누락, 쉼표 오류, 괄호 불일치 등)가 있으면 파싱이 실패합니다. 이 도구에 JSON을 붙여넣으면 오류 위치와 원인을 즉시 확인할 수 있습니다.
JSON 미니파이(압축)란?
공백·줄바꿈을 제거해 JSON을 최소 크기로 압축하는 것입니다. API 전송 시 데이터 크기를 줄이거나 코드에 인라인으로 삽입할 때 사용합니다.
JSON과 JavaScript 객체의 차이는?
JavaScript 객체는 키에 따옴표 없이 쓸 수 있고 주석도 허용됩니다. JSON은 표준 데이터 교환 형식으로 키에 반드시 큰따옴표가 필요하고 주석이 불가합니다.
JSON 들여쓰기 몇 칸이 표준인가요?
JSON 자체에는 들여쓰기 표준이 없습니다. 일반적으로 2칸 또는 4칸 공백을 사용하며, 일부 프로젝트는 탭을 사용합니다. 가독성을 위해 일관된 스타일을 선택하세요.
JSON에서 허용되는 값 타입은?
문자열(string), 숫자(number), 불리언(true/false), null, 배열([]), 객체({})입니다. undefined, 함수, Date 객체 등은 JSON에서 지원되지 않습니다.
JSON 주석을 사용할 수 있나요?
표준 JSON에서는 주석(// 또는 /* */)을 지원하지 않습니다. 주석이 필요하다면 JSONC(JSON with Comments)나 JSON5 형식을 사용하거나, _comment 키로 주석을 대신할 수 있습니다.
JSON trailing comma 오류란?
Trailing comma는 배열이나 객체의 마지막 항목 뒤에 쉼표가 붙는 것입니다. JavaScript에서는 허용되지만 JSON 표준에서는 오류입니다. 이 도구로 검증하면 해당 오류를 바로 확인할 수 있습니다.
